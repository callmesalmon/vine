VERY INTUITIVE EDITOR TUTORIAL

This is the help/tutorial/tutor for the Very INtuitive Editor, the mostest inuitivest fork of kilo.
While editing files at surface level may seem relatively simple, as VINE provides a very GNANO-like
interface, it actually has quite a lot of depth to it if you learn everything about the editor.

1. EXITING
Every terminal editor has to be excruciatingly hard to exit. In VINE's case, you may quit with ``C-x C-c``,
which EMACS users will find familiar. If you want to test it out you may, however note that this will obviously
exit you out of the tutorial.

2. BASIC EDITING
This'll be no long section, as it is extremely simple. There are no modes in VINE. You just write text
and move around with the arrow keys. Try it out! It ain't hard!

3. SAVING AND OPENING
While you can open a file using ``vine [filename]``, you might find it useful to open a file while in the editor.
Therefore, VINE provides ``C-x C-f``. As VINE does not have "multiple buffers" in the way that other editors
might, make sure to save your changes before opening a new file. You save using ``C-x C-s``.

Now that you've read this section, close this tutorial and open a new file. Write something into it, save the
contents, and then open a new file from within the editor. Do this until you feel comfortable.

4. MOTIONS
You can edit files in VINE fairly fast using the (naturally) EMACS-like motions. You can move across words
using (fwd) M-f and (bwd) M-b. Try it out! Move across this section using M-f and M-b! Note that a "word"
in VINE is EXCLUSIVELY separated by spaces, i.e "mary,had,a,little,lamb" is ONE WORD.

You may also use the M-g shortcuts. M-g j moves to the start of a line while M-g k moves to the end.
M-g s move to the start of the FILE, and M-g e moves to the end of the file. Additionally, there is
the M-g g shortcut, which moves to a specified line.

ADDITIONALLY, you may use C-s to find [query] in a file.

5. DELETING
You may delete the previous character using BACKSPACE, like usual, but you may also delete the NEXT
character using C-d. You may also use C-k do delete an entire line.

6. COMMANDS
You can run system commands with C-c C-s. Note that this only prints if command succeeded or not,
and not the command output. You can also run VINE configuration commands (see section 7) using
C-c C-c.

7. BASIC CONFIGURATION
The VINE configuration operates using a config file, ~/.vinerc. Comments may be added, starting with " like in
VimScript. The following is a list of configuration options:

tab-size <int>              - Tab size
show-empty-lines <bool>     - Whether or not to show tildes on empty lines
expand-tab <bool>           - Whether or not to expand tab to [tab_size] number of spaces.
colorscheme <str>           - Set colorscheme
autopair <bool>             - Match quotes and braces.
colr-{x} <color>            - Set color of highlight-fields.

List of colorschemes (must be in quotes):

``default`` - The default colorscheme
``kilo``    - The good ol' original kilo colorscheme
``slate``   - Vim's ``slate`` colorscheme, basically.
``evening`` - Vim's ``evening`` colorscheme.
``elflord`` - Vim's ``elflord`` colorscheme.
``quiet``   - A quiet dark theme.
``light``   - Psychopathic light theme.

8. COLORSCHEMES
As somewhat mentioned above, VINE *does* provide a very simple way to create and modify colorschemes. Simply put,
``colr-{field} = {value}``.

The fields are:
- colr-keyword1
- colr-keyword2
- colr-comment
- colr-number
- colr-string
- colr-find
- colr-default
- colr-statusbar
- colr-linenum
- colr-bg

And the color values are:
- red
- green
- blue
- purple
- cyan
- white
- black
- grey
Additionally, all color values except for white, black and grey have their own bright variations,
prefixed with "b_", so bright red becomes ``b_red`` and so on...
